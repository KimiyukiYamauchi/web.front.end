# 期末課題作成

## 1. 要件

以下の要件をすべて満たす Web アプリ を作成してください。

使用技術

- Next.js (フレームワーク)
  - プロジェクト作成時の設定
    - TypeScript
    - ESLint
    - App Router
  - スタイル設定
    - globals.css
    - CSS modules(Scoped CSS)
  - ルーティング(トップページ、下層ページ)
- microCMS (データ保存)
- Github(ソース管理)
  - README.md ファイルに、作成する Web アプリの 機能仕様 を記載する
  - 既存の README.md は必要に応じて README2.md にリネームしてもよい
- Vercel（デプロイ・公開サーバ）

## 2. スケジュール

### 1. 公開内容の提出 — 11/11（火）まで

Slack に以下の情報を投稿してください。

- 氏名(学生番号)
- 作成するアプリの機能
- プロジェクト名
- Github の URL
- Vercel の URL

投稿例：

```
- 山内公之
- 検定の問題サイト(ITF+、SEA/J)
- quizsite
- https://github.com/KimiyukiYamauchi/quizsite
- https://quizsite-beta.vercel.app/

```

### 2. 開発期間 — 11/12（水）〜 11/26（水）

- 日次で成果物を GitHub に push すること  
  → この更新状況を 授業態度の評価 に反映します

### 3. アプリ説明会 — 11/27（木）

発表内容（説明のポイント）：

- 機能の説明
- ページ構成の説明
- ディレクトリ構成の説明
- microCMS にアクセスする処理の説明

## 3. 評価のポイント

- アプリが「1. 課題要件」をすべて満たしているか
- アプリ説明会で、上記の説明ポイントが明確に説明されているか
